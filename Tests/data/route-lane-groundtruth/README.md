# Route lane "ground truth" 

## Motivation

Suggested route lanes are evaluated by slightly complex and fragile heuristics.
Small change that improves one junction may cause a regression in another one. 
This data set is intended to be used as a "ground truth" for testing and improving 
the lane suggestion heuristics. Every route in the dataset is a real-world route 
that has been manually verified. But may contain errors. When your change 
in the heuristics generate a "better" lane suggestion for a route in the dataset, 
please update route in the dataset.

There is RoutePostprocessorTest with basic heuristics sanity checks, but it is impractical
for checking regressions in real-world routes.

## Verification

Ground truth database is generated by `LaneEvaluation` test tool. 
To verify that your change does not cause regressions in the dataset, 
run the following commands:

```bash
# "build" is build directory
DB=path-to-country-db

./Tests/scripts/recompute-routes.sh \
    ./build/Tests/LaneEvaluation \
    $DB \
    Tests/data/route-lane-groundtruth/cz \
    "./groundtruth-check"

./build/Tests/LaneEvaluation compare \
    --images "./groundtruth-check" \
    --stylesheet stylesheets/standard.oss \
    --database $DB \
    Tests/data/route-lane-groundtruth/cz \
    "./groundtruth-check"
```
